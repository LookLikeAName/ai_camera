# AI Camara: Image Reconstruction via Gemini

## Project Overview
AI Camara is a static web application that performs "image-to-text-to-image" reconstruction. The user uploads an image, a vision model describes it in exhaustive detail, and that description is used as a prompt for an image generation model to recreate the scene.

## Core Features
1.  **Direct API Integration**: Communicates with the Google Gemini API directly from the browser using RESTful requests.
2.  **API Key Management**: 
    -   Support multiple API keys in `localStorage`.
    -   Ability to add, delete, and toggle an "active" key.
    -   Only one key is active at a time (activating one disables others).
3.  **Vision Analysis**: Uses `gemini-3.1-flash-lite-preview` to generate a high-fidelity, detailed description of an uploaded image.
Refer: https://ai.google.dev/gemini-api/docs/gemini-3
4.  **Image Generation**: Uses the prompt generated by the vision model to create a new image via `gemini-3.1-flash-image-preview`. 
Refer: https://ai.google.dev/gemini-api/docs/image-generation
5.  **Reconstruction Workflow**:
    -   User uploads an image or captures one (Phase 2).
    -   Automatic flow: Vision description is generated and immediately fed into the Image Generation model.
    -   The intermediate prompt (AI Vision description) is logged to the browser console for debugging.
    -   Displays the final AI-generated image as the primary output.
6.  **Static Deployment**: Compiled as a single-page application (SPA) suitable for GitHub Pages.

## Tech Stack
-   **Framework**: React (TypeScript)
-   **Build Tool**: Vite
-   **Styling**: Vanilla CSS (Modern CSS features like Flexbox/Grid)
-   **API**: Google Gemini REST API (v1beta)
-   **Deployment**: GitHub Pages

## UI/UX Requirements
-   **"Camera-like" Interface**: An aesthetic inspired by camera viewfinders, shutter buttons, and lens elements.
-   **Phase 1**: File upload focus.
-   **Phase 2**: Direct camera stream integration.
-   **No History**: The app only displays the current result; no session history is stored for image pairs.
-   **Responsive**: Works on mobile and desktop.
-   **Loading States**: Visual feedback during the long-running AI processing steps.
